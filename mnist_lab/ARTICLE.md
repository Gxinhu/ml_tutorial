# 用 uv + PyTorch 从零实现手写数字识别

## 1. 这个项目想解决什么问题

我们要做的是一个最经典的图像分类任务：给模型一张手写数字图片，让它判断这张图片是 `0` 到 `9` 里的哪一个数字。

这个任务很适合初学者，因为：

- 输入很直观：一张灰度图
- 输出很简单：10 个类别
- 模型结构也比较标准：卷积层提特征，全连接层做分类

## 2. 为什么用卷积神经网络

对于图片来说，相邻像素之间是有空间关系的。

卷积层的作用可以简单理解成：

1. 用一个小窗口在图像上滑动
2. 每次只看局部区域
3. 把局部区域里的边缘、拐角、笔画等信息提取出来

这样模型就不会把图片当成普通的一维数字表格，而是会保留图像的结构信息。

## 3. 代码结构是怎么设计的

为了让代码更容易读，我们把职责拆成了几个类：

- `TrainingConfig`
  统一管理批大小、学习率、训练轮数、保存路径等参数

- `MNISTDataModule`
  负责下载数据集、做预处理、创建 `DataLoader`

- `DigitCNN`
  负责定义卷积神经网络结构

- `ModelTrainer`
  负责训练、验证、保存模型、画训练曲线

- `DigitPredictor`
  负责加载训练好的模型，并对单张图片做预测

## 4. 数据是如何进入模型的

在 `data.py` 里，核心流程是：

1. 下载 MNIST 数据集
2. 把图片转换成 `Tensor`
3. 做标准化
4. 用 `DataLoader` 按批次读入数据

这样在训练时，每次拿到的是一批图片和一批标签，而不是一张一张地读。

这里可以顺手记住几个基础名词：

- `pixel`：图片里的一个小格子
- `channel`：图像通道，灰度图只有 1 个通道
- `batch`：一次送进模型的一批图片
- `epoch`：把整个训练集完整看过一遍

这里专门写一个 `MNISTDataModule` 是有意义的。

因为数据准备通常会包含几类固定工作：

- 下载数据集
- 定义统一的预处理
- 构建训练集和测试集
- 包装成 `DataLoader`

如果把这些逻辑直接写进训练循环里，代码会很快变乱。拆成 `DataModule` 以后，训练器就可以只关注“模型怎么训练”，而不用再管“数据怎么读进来”。

更规范一点的话，还应该把数据分成三部分：

- `train`
- `val`
- `test`

其中：

- `train` 用来更新参数
- `val` 用来在训练过程中观察模型是否开始过拟合
- `test` 只在训练结束后做最终评估

这样就不会在训练过程中反复“偷看”测试集。

## 5. 模型结构怎么理解

`DigitCNN` 里用了两层卷积：

- 第一层：从原始像素里提取基础特征
- 第二层：把前面的基础特征继续组合

之后用：

- `Flatten` 把特征图拉平
- `Linear` 把特征映射到 10 个类别

最后输出的是 10 个类别分数，不是直接输出最终答案。最终答案通过 `argmax` 取分数最大的类别得到。

另外，这里层的顺序也值得专门记一下：

1. `Conv2d`
   先提取局部特征，比如边缘和笔画

2. `ReLU`
   给网络加入非线性，让模型能表示更复杂的关系

3. `MaxPool2d`
   在已经提取出的特征上做下采样，缩小尺寸，同时尽量保留最强响应

所以常见顺序是：

```text
Conv2d -> ReLU -> MaxPool2d
```

这个顺序的直觉是：

- 先找到特征
- 再用激活函数筛出有效响应
- 最后再压缩尺寸

## 6. 训练循环的核心三步

`trainer.py` 里最重要的逻辑就是这三步：

1. 前向传播
   把图片送进模型，得到预测结果

2. 反向传播
   根据损失函数计算梯度

3. 参数更新
   用优化器根据梯度更新模型参数

对应的关键代码是：

```python
logits = self.model(images)
loss = self.criterion(logits, labels)

self.optimizer.zero_grad()
loss.backward()
self.optimizer.step()
```

这里：

- `criterion` 是交叉熵损失
- `backward()` 负责反向传播
- `step()` 负责真正更新参数

训练时每个 epoch 后，更推荐输出的是：

- `train_loss`
- `train_acc`
- `val_loss`
- `val_acc`

而不是每个 epoch 都去盯测试集。

## 7. 学习率在这里起什么作用

学习率控制的是每次参数更新时走多大一步。

- 学习率太大：训练容易不稳定
- 学习率太小：训练会很慢

所以它决定的是“更新幅度”，而不是“更新方向”。
更新方向来自梯度，更新幅度来自学习率。

## 8. 如何做预测

训练完以后，可以用 `DigitPredictor` 对单张图片做推理。

流程是：

1. 读取图片
2. 转成灰度图
3. 调整成 `28x28`
4. 做和训练时一样的张量转换与标准化
5. 输入模型并输出预测类别

这样可以帮助你把“训练代码”和“实际使用模型”两个阶段分开理解。

## 9. 推荐的学习顺序

如果你是第一次接触 PyTorch，建议按这个顺序看代码：

1. 先看 `config.py`
   先搞清楚项目里有哪些超参数

2. 再看 `model.py`
   先理解模型结构，不要一上来就钻训练细节

3. 然后看 `data.py`
   看数据是怎么变成模型输入的

4. 最后看 `trainer.py`
   把前向传播、损失、反向传播、参数更新串起来

5. 最后读 `predict.py`
   理解模型训练好后如何真正被调用

## 10. 实操建议

第一次运行时，不要把 epoch 设置太大。

推荐先这样：

```bash
uv run python -m mnist_lab.main train --epochs 3
```

这样更容易快速看到完整流程，而不是一开始就等很久。

等你理解整个项目结构后，再去调：

- 学习率
- 卷积层通道数
- batch size
- 训练轮数

这样学习效率会高很多。

## 11. 建议先跑 inspect

如果你是第一次接触图像任务，推荐先运行：

```bash
uv run python -m mnist_lab.main inspect
```

它会告诉你：

- 一批图片的张量形状
- 一批标签的形状
- 前几个标签长什么样

同时还会把一张样例拼图保存到 `artifacts/sample_grid.png`。

这一步的作用，是先看清楚“数据到底以什么形式进入模型”，不要一上来只盯着卷积层代码。

## 12. 如果你想先快速演示预测流程

项目里已经放了一个示例图片：

```bash
uv run python -m mnist_lab.main predict --image assets/sample_digit.pgm
```

这条命令除了打印预测数字，还会把结果图保存到：

```text
artifacts/prediction_preview.png
```

更推荐的完整顺序仍然是：

1. 先训练模型
2. 再做 evaluate
3. 最后对单张图片做 predict

因为没有训练好的权重时，预测结果本身没有意义。
